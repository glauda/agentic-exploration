# Agent Guidelines

## Committing Code

**Always ask for user consent before committing or pushing code.** Never commit, push, or make any changes to the repository without explicit user approval.

## Commit Message Format

Use the format: `prefix(scope): message`

**Prefixes:**
- `feat` - New feature
- `fix` - Bug fix
- `docs` - Documentation changes
- `refactor` - Code refactoring (no functional change)
- `chore` - Maintenance tasks (dependencies, configs, etc.)
- `test` - Adding or updating tests

**Examples:**
- `feat(auth): add login validation`
- `fix(api): handle null response in user endpoint`
- `docs(readme): update installation instructions`
- `refactor(utils): simplify error handling`

Keep messages concise (under 72 characters). Focus on the "why" rather than the "what".
